title: Choose your path
description: Pick the shortest documentation lane for your goal.
---

| If you want to… | Start here | Why |
|---|---|---|
| Understand Turnkey end to end | [Learn](../../learn/) | Ordered chapters build one mental model at a time. |
| Complete a specific task | [Tutorials](../../tutorials/) | Each walkthrough ends in an observable artifact. |
| Integrate your own code | [External components](../../guides/external-components/) | Focuses on boundaries and entry points. |
| Diagnose a failure | [Troubleshooting](../../guides/troubleshooting/) | Routes symptoms to the right evidence. |
| Look up an exact contract | [Reference](../../reference/) | CLI, config, schemas, and release state without a lesson. |

The lanes deliberately overlap by links, not copied prose. A concept page
explains *why*; a tutorial explains *how to finish a task*; reference states
*exactly what exists*.
