<section class="tk-section" aria-labelledby="start-heading">
  <span class="tk-kicker">Source preview · 0.1.0</span>
  <h2 id="start-heading">From zero to an auditable run.</h2>
  <p class="tk-lede">
    Start locally on CPU with a small built-in path. The same component boundary
    carries forward when you connect a real detector, model provider, dataset,
    or attack.
  </p>
  <div class="tk-install">
    <code>pip install turnkey<br />turnkey init my-detector<br />cd my-detector<br />turnkey dev ./detector.py:build</code>
    <p class="tk-release-note">
      <strong>0.1.0</strong> is the functional release on PyPI. Prefer a source
      checkout when you want the full config library and development extras.
    </p>
  </div>
  <p><a href="/turnkey-site/start/install/">Read the installation status →</a></p>
</section>

<section class="tk-section" aria-labelledby="work-heading">
  <span class="tk-kicker">One harness · four jobs</span>
  <h2 id="work-heading">Run. Build. Debug. Audit.</h2>
  <div class="tk-grid">
    <article class="tk-card">
      <span class="tk-number">01 / RUN</span>
      <h3>Execute a comparable experiment</h3>
      <p>Resolve versioned inputs, run an attack/detector path, and preserve the evidence needed to inspect it later.</p>
    </article>
    <article class="tk-card">
      <span class="tk-number">02 / BUILD</span>
      <h3>Bring your own detector</h3>
      <p>Implement the public component boundary instead of forking the runner, metrics, and artifact pipeline.</p>
    </article>
    <article class="tk-card">
      <span class="tk-number">03 / DEBUG</span>
      <h3>Trace a missed case</h3>
      <p>Follow structured events and typed signals to find where selection, policy, provider, or judgment diverged.</p>
    </article>
    <article class="tk-card">
      <span class="tk-number">04 / AUDIT</span>
      <h3>Recompute, do not just trust</h3>
      <p>Cross-check cases, events, metrics, and run metadata to make unsupported success claims harder.</p>
    </article>
  </div>
</section>

<section class="tk-section" aria-labelledby="runtime-heading">
  <span class="tk-kicker">Runtime model</span>
  <h2 id="runtime-heading">A readable path from input to evidence.</h2>
  <p class="tk-lede">
    Turnkey keeps the experiment path explicit. Built-ins use registry aliases;
    external components use module or file entry points behind the same contract.
  </p>
  <div class="tk-flow" role="img" aria-label="Dataset flows through content selection, attack, policy and provider, judge, metrics, artifacts, then audit">
    dataset → selection → attack → policy/provider → judge → metrics → artifacts → audit
  </div>
  <p><a href="/turnkey-site/concepts/system-model/">Understand the system model →</a></p>
</section>
