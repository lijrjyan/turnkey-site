---
title: Security
description: Report vulnerabilities and documentation leaks privately.
---

Do not open a public issue containing a vulnerability, credential, private
dataset, or personal information. Use GitHub's private vulnerability reporting
channel when it is available.

The website is static. Runtime security issues belong to the product repository
and should identify the exact affected version or commit. Documentation leaks
in this repository should identify the affected path without repeating the
sensitive value in public discussion.
