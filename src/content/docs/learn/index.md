This is the continuous learning path. Read it in order when Turnkey is new to
you; use the rest of the site as a reference when you already know the model.

| Chapter | Outcome | Notebook |
|---|---|---|
| 1. First auditable run | Connect a command to its evidence files. | `00_quickstart.ipynb` |
| 2. Build a detector | Implement the real extension boundary with small local behavior. | `01_build_your_own_detector.ipynb` |
| 3. Debug a missed case | Trace one failure through cases and events. | `02_debug_a_missed_case.ipynb` |
| 4. Typed signals | Explain how structured detector evidence crosses the pipeline. | `03_typed_signals.ipynb` |

Each notebook is published with the source product commit and SHA-256 digest in
[`provenance.json`](/turnkey-site/notebooks/provenance.json). The prose teaches
the mechanism; the notebook gives you a compact executable checkpoint.

Start with [Chapter 1: First auditable run](./quickstart/).
