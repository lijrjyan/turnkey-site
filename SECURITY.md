# Security Policy

## Report a vulnerability

Do not open a public issue for a vulnerability, leaked credential, private
dataset, or unsafe example. Use GitHub's private vulnerability reporting for
the product repository when available, or contact the repository owner
privately through GitHub.

Include the affected page or component, impact, reproduction steps, and any
suggested mitigation. Remove secrets and personal data from screenshots and
logs. Documentation-only defects that do not expose security-sensitive details
may use a normal issue.

This site is static and does not process user data. Security claims about the
Turnkey runtime belong to the product repository and must identify the affected
version or commit.
