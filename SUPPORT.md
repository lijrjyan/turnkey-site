# Support

Use the documentation search and [Troubleshooting](https://lijrjyan.github.io/turnkey-site/guides/troubleshooting/)
before opening a report.

Use the product repository for runtime bugs, detector behavior, packaging, and
feature requests. Use the site repository for incorrect examples, broken links,
navigation, accessibility, or website build/deployment problems.

A useful report includes the exact product commit or version, site page URL,
smallest reproduction, expected result, actual result, and sanitized error
output. Do not attach credentials, private prompts, internal datasets, personal
data, or unredacted artifact bundles. Security-sensitive reports follow
[SECURITY.md](SECURITY.md), not public issues.

The project does not currently promise a response-time service level. Community
support and maintenance are best effort.
